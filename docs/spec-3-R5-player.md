# 3-R5 実行モード（ステッププレイヤー）＋書き出しHTML同梱 — 確定仕様

2026-07-14 確定（モック承認済み。モックは開発セッションのアーティファクト参照）。
ロードマップ 3-R5 の実装方針。**費用ゼロ方針**（追加依存なし・単一HTML維持）で実装する。

## 目的

手順書を「1ステップ全画面表示・完了チェック・キーボードで次へ」で進める実行モードを
レンダラーに追加する。所要時間フィールド（分）を標準時間として実測時間と比較表示する。
書き出した単体HTMLにも同じ実行モードを同梱し、受け取った人がブラウザだけで
ステップ実行できるようにする。

## 確定した決定事項（2026-07-14 ユーザー承認・既定案どおり）

1. **完了チェックは本体のチェック状態と同期する。** 実行モードでの「完了して次へ」は
   `item.done = true` を通常の commit（Undo 可能・進捗/一括リセットと連動）で反映する。
2. **実測時間は保存しない**（表示のみ）。実施記録（点検ログ）は当面作らない方針
   （ロードマップ前提5）と整合。
3. **開始単位はリスト全体＋セクション単位の両方。** どちらも「未チェックの最初の
   ステップ」から再開する（全部チェック済みなら先頭から）。
4. **スキップはチェックなしで次へ進む。** サマリに「スキップ」として表示し、
   本体のチェックは付けない。
5. **画像が複数ある項目は1枚目を大きく表示し、切替可**（サムネイル選択）。
6. **書き出しHTML側のチェックは従来どおりメモリのみ**（単体文書モードは
   MemoryAdapter のためリロードで消える。localStorage 保存はしない）。
7. **見出し・区切りは読み飛ばす。** 本アプリのモデルでは全項目がチェック可能なため、
   ステップ＝全セクションの全項目（セクションは文脈表示に使う）。

## 画面構成（モックどおり）

- **プレイヤー**: 全画面オーバーレイ。上部=リスト名・セクション名・「ステップ n / N」・
  終了ボタン・進捗バー。中央=左に画像（複数はサムネイルで切替）、右に手順文・
  リッチ本文・メモ・実測/標準時間。下部=「← 戻る」「✓ 完了して次へ」「スキップ →」
  とキーボードヒント。
- **完了サマリ**: 最終ステップの後に表示。完了/スキップ数・合計実測・合計標準の
  統計と、ステップ別の結果表（標準超過は強調）。「リストに戻る」「もう一度実行」。
- **キーボード**: `Space`/`Enter`=完了して次へ、`←`=戻る、`→`=スキップ（チェックなしで次へ）、
  `Esc`=終了。
- **導線**: エディタのツールバーに「▶ 実行」（項目0件なら無効）。各セクション見出しに
  「▶」（そのセクションだけ実行）。

## 実装方式

- レンダラー（index.html）内に完結。プレイヤーは `.player-overlay`（fixed 全画面、
  デザイントークン使用、`@media print` では非表示）。
- 純関数（テスト対象・`window.__test__` に公開）:
  - `playerSteps(checklist, sectionId?)` — 実行対象ステップの平坦化
  - `playerStartIndex(steps)` — 開始位置（最初の未チェック。無ければ 0）
  - `formatPlayerTime(seconds)` — `m:ss` 表示
  - `playerSummary(steps, results)` — 完了/スキップ/合計実測/合計標準の集計
- 実測時間は表示中のステップに 1 秒刻みで加算（`setInterval`）。サマリ表示中は停止。
  開始時点でチェック済みだったステップは「完了（0:00）」として扱う。
- 画像は既存の `resolveImageSync` ＋ `data-img-ref` パターンで非同期解決
  （Electron のファイル参照・ブラウザの dataURL の両対応）。
- **書き出しHTMLへの同梱は自動で成立する**: `buildStandaloneHtml` はアプリの
  `<script>` 全体を埋め込むため、本体に実装した実行モードがそのまま載る。
  単体文書モードは MemoryAdapter のため決定6も自動で満たす。
