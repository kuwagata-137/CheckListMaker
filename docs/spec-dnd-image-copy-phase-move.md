# 仕様: 画像のD&Dコピー / 手順のフェーズ間D&D移動 / 画像挿入の既定フォルダ

エディタの操作性を上げる3点。着手前にユーザーと確定した仕様（2026-07-17）。

## 1. 隣（任意）の手順の画像をD&Dでコピー

- 対象: エディタの手順（`.item` / `.step`）に添付されたサムネイル（`.thumb`）。
- 操作: サムネイルを掴んで**別の手順**の上へドロップすると、その手順に画像が**追加**される。
- 範囲: 「隣」に限らず**任意の手順間**でコピー可（同一フェーズ/別フェーズを問わない）。
- コピーであって移動ではない: **元の手順にも画像は残る**。
- 参照は**共有**する: Electron のファイル画像モードでは、原寸/サムネの個別ファイルは
  複製せず、同じ参照（`img:<uuid>`）をコピー先にも持たせる。ファイルの実削除は
  起動時 GC が「どの手順からも参照されなくなったとき」だけ行うため、共有しても安全。
  片方の手順で画像を編集すると、`replaceItemImage` が新しい参照に差し替わるため、
  もう片方には影響しない（編集時点で実体が分岐する）。
  - 編集メタデータ（`imageEdits`）は取り違え防止のためコピー時に**複製**する
    （小さな JSON。実体の重い画像ファイルだけを共有する）。
- 同一手順へのドロップは何もしない（「手順間」コピーのため）。
- ブラウザ単体モードでは `images[i]` が dataURL 実体のため、コピーは dataURL の複製に
  なる（保存容量に効く）。`commitImage` で保存失敗時は取り消し＋通知する。

## 2. 別フェーズの手順をD&Dで移動（ドロップ判定の拡張）

- 既存の並べ替え（`attachDragAndDrop` / `moveItem`）はフェーズ間移動に対応済みだが、
  ドロップ判定 `itemDropAt` が `.item` と `.items`（手順リスト）しか拾わないため、
  **フェーズ見出し・空フェーズ・手順のすき間**に落とすと移動できなかった。
- 対応: `itemDropAt` に**フェーズ（`.section`）フォールバック**を追加する。
  - `.item` / `.items` に当たらず `.section` 上にあるときは、そのフェーズの手順リスト
    （`.items`）を対象にする。
  - リスト上端より上（見出し付近）にポインタがあれば**先頭**（index 0）、それ以外は
    **末尾**へ挿入する。空フェーズは index 0。
- 挿入インジケーターの表示位置は従来同様に算出する。

## 3. 画像挿入ボタンの参照先を既定フォルダに

- 現状は Web の `<input type="file">` で初期フォルダを指定できない（ブラウザ任せ）。
- Electron では**ネイティブダイアログ**（`dialog.showOpenDialog`）に切替え、
  `defaultPath` に `Pictures\CheckListMaker`（＝スクショ保存先 `screenshotDir()`）を指定する。
  - フォルダが無ければ**自動作成**（`fs.mkdirSync(recursive)`）してから開く。
  - 選択結果は dataURL でレンダラーへ返し、従来の追加フロー（`storeImagePair` →
    `addItemImage`）に載せる。
- ブラウザ単体モード（`window.fileAPI.pickImage` 不在）は従来の `<input type=file>` に
  フォールバックする。
- IPC: `image:pickFile`（main）/ `window.fileAPI.pickImage()`（preload）。

## テスト観点

- `copyItemImage`: 参照共有でコピー先に追加され、元は残る。`imageEdits` は複製。
- `itemDropAt` 相当のフェーズ移動は UI 依存のため、`moveItem` の純関数テスト（既存）で
  フェーズ間移動の不変条件を担保。ダイアログ/DnD の配線は実機検証で確認する。
