# 録画＝クリック連動スクリーンショット機能 — 確定仕様

> 本仕様に沿った実装を **main に統合済み**（5つの確定項目を反映）。残るは実機での
> 動作検証（マルチモニタの display 対応付け・各OSの権限・Linux の撮影除外）。

## 背景・目的

操作手順を記録したいとき向けの機能。**録画ボタン（⏺）を押すと、以降クリックする
たびに画面のスクリーンショットを自動保存**する（動画としては記録しない）。録画中は
画面右上に専用の **ガジェット**（経過時間・撮影枚数・最後のプレビュー・停止ボタン）を
表示し、このガジェット自体はスクリーンショットには写らない。

## 採用アーキテクチャ

CheckListMaker は本来 **単一 `index.html`・依存ゼロ・ブラウザで動く純Webアプリ**。
しかしブラウザのサンドボックスでは以下が**原理的に不可能**:

- 他アプリのクリック（OSレベルのグローバルマウスイベント）を検知できない。
- 任意のローカルフォルダ（アプリのあるディレクトリ）へ自動で書き込めない。
- 画面キャプチャは `getDisplayMedia()` で毎回手動許可＋共有インジケータ表示が必須。

→ 全要件を忠実に満たすため **A) Electron 化** を採用する。既存 `index.html` を一切
書き換えず「画面（renderer）」として再利用し、録画機能だけをメインプロセス側の
ネイティブ機能で実装する。

> B案（ブラウザ維持・範囲縮小版）／C案（Tauri 化）は**別ブランチ／別セッション**で
> 後日検討する。概要は本書末尾を参照。

## 確定仕様

| 項目 | 確定内容 |
|------|----------|
| 撮影トリガ | **左クリック＋右クリックのみ**（中クリック/スクロール押し込みは除外） |
| 自アプリ除外 | CheckListMaker の**メイン窓・ガジェット窓のクリックは撮影しない** |
| 撮影範囲 | **クリックしたモニタ**（マルチモニタ対応） |
| 撮影タイミング | **即時（0ms）** |
| 保存先 | ユーザーの **「ピクチャ」直下の `CheckListMaker`** フォルダ（無ければ自動作成。Windows なら `%USERPROFILE%\Pictures\CheckListMaker`）。取得不能な環境ではアプリのあるディレクトリ直下へフォールバック。**2026-07-13 更新（2-R1）**: その配下に**録画1回ごとのセッションフォルダ** `<リスト名>_<yyyymmdd>_<hhmmss>/` を作って保存する（詳細は `docs/spec-2-R1-session-format.md`） |
| 録画終了時 | 停止時に**そのセッションのフォルダをエクスプローラー（OS のファイルマネージャ）で自動的に開く**（2-R1 更新。0枚なら空フォルダを削除して親フォルダを開く） |
| ファイル名 | **セッションフォルダ内の `001.png`＋`001.json`（クリック情報のメタデータ併記）**（2-R1 更新。リスト名はフォルダ名が担う。旧方式はフラットな `<リスト名>_<yyyymmdd>_<番号>.png` で、既存ファイルはそのまま残る） |
| 連番 | **セッション内で 1 から・3桁ゼロ埋め**（2-R1 更新。フォルダがセッションごとに分かれるため、旧方式の「同名・同日ファイルの最大番号+1」走査は廃止） |
| 画像形式 | **PNG** |
| クリックマーカー | **クリック位置に赤い円を重ねて保存**（手順書として操作箇所を明示） |
| ガジェット | 経過時間・撮影枚数・最後のプレビュー・録画停止ボタン。`setContentProtection(true)` で撮影対象から除外 |
| 停止 | ガジェットの停止ボタン、または録画ボタンの再押下 |

### 補足
- ガジェットの「撮影枚数」は**当該録画セッションで撮った枚数**を表示（ファイル名の連番とは独立）。
- クリックマーカーの座標変換は Electron の `display.bounds`（DIP）と `display.scaleFactor` を用い、
  物理相対 x = `physX - bounds.x * scaleFactor`（y も同様）で求める。
- マーカー合成は**新規依存を避け**、メイン窓 renderer の canvas を
  `mainWin.webContents.executeJavaScript()` 経由で利用（失敗時は素の画像を保存）。

## 実装前に確定した詳細仕様（決定事項）

5つの未反映点を実装するにあたり、仕様書の曖昧さを潰すために確定した詳細。
数値はすべて**初期値**であり、実機検証で微調整してよい。

| 論点 | 確定した挙動 |
|------|------------|
| **連番のリセット** | **日付ごとに 1 から振り直す**（`<名前>_<同日付>_*.png` の最大番号+1）。日をまたぐと日付が変わるため上書きは起きない。 |
| **赤い円マーカーの形状** | **中抜きリング**（塗り潰さない）。初期値: 半径 20px・線幅 4px・色 `#ef4444`・不透明度 1.0。リング状なのでクリック対象のアイコン等が隠れない。 |
| **マーカーのサイズ基準** | 物理ピクセル基準で描画し、**モニタの `scaleFactor` に比例**させる（HiDPI で小さくなりすぎないように）。 |
| **ダブルクリック／高速連打** | **短時間デバウンスで1枚に集約**（初期値 300ms）。同一位置近傍での連続クリックは1枚にまとめ、手順書の枚数が膨らまないようにする。 |
| **右クリックと撮影タイミング** | 即時（0ms）撮影＝**クリック直後の結果画面**を撮る。右クリックで開くコンテキストメニューは描画前のため**写さない**（写すことを目的としない）。 |
| **ドラッグ操作** | ドラッグ（押下→移動→離す）は**撮影対象としない**。`click`（同一地点での押下・離上）のみを撮影トリガとする。 |
| **メイン窓の除外範囲** | CheckListMaker の**メイン窓全体**を撮影除外（ガジェット窓に加えて）。用途は「別アプリの操作記録」のため、チェックリスト自身の操作は記録しない。録画ボタン押下も当然撮らない。 |
| **失敗・権限拒否時の通知** | 権限拒否（macOS の画面収録／アクセシビリティ）・グローバルフック開始失敗・キャプチャ失敗を検知したら、**ガジェット上に赤字で警告を表示**する（「録画したのに0枚」を防ぐ）。`console.error` だけにしない。 |
| **Linux の撮影除外** | `setContentProtection` が X11 で効くか確実に検出できないため、**Linux では常時「撮影直前にガジェットを隠す→撮影→戻す」**方式に倒す（チラつきより確実性を優先）。Windows/macOS は `setContentProtection` を使う。 |

### マルチモニタ対応の実装方針（要・実機検証）

「クリックしたモニタを撮影」は次の対応付けで実装する。**`screenshot-desktop` のディスプレイ
列挙順と Electron の `display.id` は一致保証がない**ため、実装前に実機で突き合わせ検証する。

1. `uIOhook` の物理クリック座標から、`screen.getDisplayNearestPoint()` で対象 Electron ディスプレイを特定。
2. `screenshot.listDisplays()` の結果と Electron ディスプレイの `bounds`／`id` を突き合わせ、撮影対象を `screenshot({ screen })` で指定。
3. マーカー座標は**撮影したモニタの `bounds` 基準の相対座標**へ変換（既存の `display.bounds`・`scaleFactor` を用いた式を流用）。

## 想定ファイル構成（実装時）

```
package.json   Electron アプリ定義・依存・スクリプト
main.js        メインプロセス（録画制御・マウス監視・画面キャプチャ・保存・ガジェット）
preload.js     contextBridge で window.recorderAPI を安全に公開
gadget.html    録画中オーバーレイ（経過時間・枚数・プレビュー・停止）
index.html     アプリ本体（ブラウザでも Electron でも同一。録画ボタンと連携処理を追記）
```

依存: `electron`, `screenshot-desktop`, `uiohook-napi`（＋ `@electron/rebuild`）。
セキュリティ: `contextIsolation:true` / `nodeIntegration:false`、IPC は preload の
`contextBridge` のみで公開。

## 実装状況

確定仕様に沿って **5項目すべてを実装済み**（`main.js` を中心に、`preload.js`・`gadget.html`）:

1. ✅ 撮影トリガを左＋右に限定（中・サイドボタン除外。`mousedown`/`mouseup` でドラッグも除外）。
2. ✅ メイン窓のクリックも除外（`isOnOwnWindow` でメイン窓・ガジェット窓を除外）。
3. ✅ クリックしたモニタを撮影（`getDisplayNearestPoint` → `screenshot({ screen })`）。
4. ✅ 連番を「日付ごとに振り直し」へ（既存ファイル走査で最大番号+1）。
5. ✅ クリック位置に赤い中抜きリングを合成（renderer の canvas 経由・失敗時は素画像）。

あわせて確定した詳細（デバウンス・失敗時のガジェット警告・Linux の撮影直前隠し）も実装済み。

### 追加改修（UI・保存先まわり）

上記の基本機能に対し、実運用のフィードバックから以下を追加で反映した:

1. ✅ **保存先をユーザーの「ピクチャ」内 `CheckListMaker` フォルダへ変更**（`app.getPath('pictures')`）。
   以前はアプリのあるディレクトリ直下の `00_スクリーンショット` だった。
2. ✅ **録画ボタンのデザインを一般的な録画マークへ**（黒い四角の中心に赤い丸の SVG アイコン）。
   以前は `⏺` 絵文字で分かりにくかった。
3. ✅ **録画停止時に保存先フォルダをエクスプローラーで自動的に開く**（`shell.openPath`）。
4. ✅ **他アプリ（Excel 等）のクリックが撮れない問題を修正**。`isOnOwnWindow` はメイン窓の
   座標だけで除外していたため、他アプリを最大化して録画するとメイン窓と座標が重なるクリックが
   誤って除外されていた。**メイン窓はフォーカス（前面）にあるときだけ除外**する条件へ変更し、
   背後に隠れているメイン窓の座標上のクリックは撮影するようにした。

> 未確定の技術リスクとして、`screenshot-desktop` の display 列挙順と Electron `display.id`
> の対応付けは実機検証が必要（現状は `getAllDisplays` の index で対応付け、失敗時はフォールバック）。

## 検証手順（実装後、実機での確認が必要）

1. `npm install && npm run rebuild`（ネイティブモジュールを Electron ABI で再ビルド）。
2. `npm start` で起動 → 既存チェックリスト画面が従来通り表示される。
3. リストを開いて録画ボタン押下 → ガジェット出現・タイマー作動。
4. 他アプリを**左/右クリック** → `00_スクリーンショット/<リスト名>_<日付>_<番号>.png` 生成、
   ガジェットの枚数・プレビュー更新。**中クリック・自アプリのクリックでは撮れない**。
5. クリック位置に**赤い円マーカー**、**クリックしたモニタ**が撮れていること。
6. 連番が**同日同名で続き番号**になること（止めて再開しても上書きなし）。
7. スクショ画像にガジェットが**写っていない**こと。
8. 停止ボタンで録画終了・ガジェット消滅・ボタン状態復帰。
9. 素のブラウザで `index.html` を開き、従来機能が壊れていない（録画ボタンは無効）こと。

> 注: ヘッドレス環境ではグローバルマウスフック・実画面キャプチャ・`setContentProtection`
> の最終確認はできない。Windows/macOS の実機で手動検証する。
> macOS は「画面収録」「アクセシビリティ」の許可、Linux は X11 系ライブラリ・`scrot`/
> ImageMagick が必要な場合がある。`setContentProtection` は Linux/X11 で効かないことがあるため、
> **Linux では常時「撮影直前にガジェットを隠す→撮影→戻す」方式**に倒す（上記「決定事項」参照）。

## 代替案（別ブランチで検討）

- **B) ブラウザ維持・範囲縮小版**: 単一HTMLを保つ。`getDisplayMedia()` で画面取得、
  File System Access API でユーザーが選んだフォルダに保存。トリガは手動キャプチャ/
  ホットキー（他アプリのクリック検知は不可）。Chromium 限定・`http(s)` 必須。要件は一部のみ充足。
- **C) Tauri 化**: A と同等の実現範囲で配布が軽量。ただし Rust ツールチェーンが必要。
