# 録画＝クリック連動スクリーンショット機能 — 確定仕様

## 背景・目的

操作手順を記録したいとき向けの機能。**録画ボタン（⏺）を押すと、以降クリックする
たびに画面のスクリーンショットを自動保存**する（動画としては記録しない）。録画中は
画面右上に専用の **ガジェット**（経過時間・撮影枚数・最後のプレビュー・停止ボタン）を
表示し、このガジェット自体はスクリーンショットには写らない。

## 重大な前提（採用アーキテクチャ）

CheckListMaker は本来 **単一 `index.html`・依存ゼロ・ブラウザで動く純Webアプリ**。
しかしブラウザのサンドボックスでは以下が**原理的に不可能**:

- 他アプリのクリック（OSレベルのグローバルマウスイベント）を検知できない。
- 任意のローカルフォルダ（アプリのあるディレクトリ）へ自動で書き込めない。
- 画面キャプチャは `getDisplayMedia()` で毎回手動許可＋共有インジケータ表示が必須。

→ 全要件を忠実に満たすため **A) Electron 化** を採用する。既存 `index.html` を一切
書き換えず「画面（renderer）」として再利用し、録画機能だけをメインプロセス側の
ネイティブ機能で実装する。

> B案（ブラウザ維持・範囲縮小版）／C案（Tauri 化）は**別ブランチ／別セッション**で
> 後日検討する。概要は本書末尾を参照。

## 確定仕様

| 項目 | 確定内容 |
|------|----------|
| 撮影トリガ | **左クリック＋右クリックのみ**（中クリック/スクロール押し込みは除外） |
| 自アプリ除外 | CheckListMaker の**メイン窓・ガジェット窓のクリックは撮影しない** |
| 撮影範囲 | **クリックしたモニタ**（マルチモニタ対応） |
| 撮影タイミング | **即時（0ms）** |
| 保存先 | アプリのあるディレクトリ直下の **`00_スクリーンショット`**（無ければ自動作成） |
| ファイル名 | **`<リスト名>_<yyyymmdd>_<番号>.png`**（リスト名＝開いているチェックリストのタイトル。未選択時は「記録」） |
| 連番 | **同日・同名の続き番号**（既存ファイルの最大番号+1。日をまたいでも上書きせず継続。ゼロ埋めなし） |
| 画像形式 | **PNG** |
| クリックマーカー | **クリック位置に赤い円を重ねて保存**（手順書として操作箇所を明示） |
| ガジェット | 経過時間・撮影枚数・最後のプレビュー・録画停止ボタン。`setContentProtection(true)` で撮影対象から除外 |
| 停止 | ガジェットの停止ボタン、または録画ボタンの再押下 |

### 補足
- ガジェットの「撮影枚数」は**当該録画セッションで撮った枚数**を表示（ファイル名の連番とは独立）。
- クリックマーカーの座標変換は Electron の `display.bounds`（DIP）と `display.scaleFactor` を用い、
  物理相対 x = `physX - bounds.x * scaleFactor`（y も同様）で求める。
- マーカー合成は**新規依存を避け**、メイン窓 renderer の canvas を
  `mainWin.webContents.executeJavaScript()` 経由で利用（失敗時は素の画像を保存）。

## ファイル構成

```
package.json   Electron アプリ定義・依存・スクリプト
main.js        メインプロセス（録画制御・マウス監視・画面キャプチャ・保存・ガジェット）
preload.js     contextBridge で window.recorderAPI を安全に公開
gadget.html    録画中オーバーレイ（経過時間・枚数・プレビュー・停止）
index.html     アプリ本体（ブラウザでも Electron でも同一。録画ボタンと連携処理を追記）
```

依存: `electron`, `screenshot-desktop`, `uiohook-napi`（＋ `@electron/rebuild`）。
セキュリティ: `contextIsolation:true` / `nodeIntegration:false`、IPC は preload の
`contextBridge` のみで公開。

## 実装状況（このブランチ）

初回コミットで Electron 化と基本フローを実装済み。**確定仕様に対する未反映点**は
別セッションで対応する（このセッションでは実装を進めない）:

1. 撮影トリガを左＋右に限定（現状は全ボタン）。
2. メイン窓のクリックも除外（現状はガジェットのみ除外）。
3. クリックしたモニタを撮影（現状はプライマリ固定）。
4. 連番を「同日同名の続き番号」に変更（現状はセッション内カウンタ）。
5. クリック位置の赤い円マーカー描画を追加（現状は未実装）。

## 検証手順（実機での確認が必要）

1. `npm install && npm run rebuild`（ネイティブモジュールを Electron ABI で再ビルド）。
2. `npm start` で起動 → 既存チェックリスト画面が従来通り表示される。
3. リストを開いて録画ボタン押下 → ガジェット出現・タイマー作動。
4. 他アプリを**左/右クリック** → `00_スクリーンショット/<リスト名>_<日付>_<番号>.png` 生成、
   ガジェットの枚数・プレビュー更新。**中クリック・自アプリのクリックでは撮れない**。
5. クリック位置に**赤い円マーカー**、**クリックしたモニタ**が撮れていること。
6. 連番が**同日同名で続き番号**になること（止めて再開しても上書きなし）。
7. スクショ画像にガジェットが**写っていない**こと。
8. 停止ボタンで録画終了・ガジェット消滅・ボタン状態復帰。
9. 素のブラウザで `index.html` を開き、従来機能が壊れていない（録画ボタンは無効）こと。

> 注: ヘッドレス環境ではグローバルマウスフック・実画面キャプチャ・`setContentProtection`
> の最終確認はできない。Windows/macOS の実機で手動検証する。
> macOS は「画面収録」「アクセシビリティ」の許可、Linux は X11 系ライブラリ・`scrot`/
> ImageMagick が必要な場合がある。`setContentProtection` は Linux/X11 で効かないことがあり、
> その際は「撮影直前にガジェットを一瞬隠す」方式へフォールバックする。

## 代替案（別ブランチで検討）

- **B) ブラウザ維持・範囲縮小版**: 単一HTMLを保つ。`getDisplayMedia()` で画面取得、
  File System Access API でユーザーが選んだフォルダに保存。トリガは手動キャプチャ/
  ホットキー（他アプリのクリック検知は不可）。Chromium 限定・`http(s)` 必須。要件は一部のみ充足。
- **C) Tauri 化**: A と同等の実現範囲で配布が軽量。ただし Rust ツールチェーンが必要。
