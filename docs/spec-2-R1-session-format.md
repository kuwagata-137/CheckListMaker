# 2-R1 録画セッション形式（画像＋メタデータ併記） — 確定仕様

ロードマップ **2-R1** の実装前に確定した仕様。撮影情報（クリック座標・時刻・対象
ディスプレイ）を PNG 保存後に捨てず、**1クリックごとに JSON サイドカーとして併記**する。
R2（UIA＋文生成）・R3（自動ズーム）・R4（取り込みウィザード）はすべてこの形式の上に載る。

## ユーザー確定事項（2026-07-13）

1. **セッションフォルダ方式へ移行する。ただし保存先の親フォルダは従来どおり
   「ピクチャ」配下（`Pictures/CheckListMaker`）を変えない。**
   録画1回ごとにサブフォルダを作り、その中に連番で保存する。
2. **クリックマーカー（赤丸）の扱いは変えない。** 既存のガジェットのオン/オフトグルを
   そのまま活かし、トグルONなら従来どおり焼き込んだ画像を保存する。素の画像の複製は
   持たない（ディスク2倍を避ける）。マーカーの有無と座標はサイドカーに記録し、
   R3 の再処理はその情報を使う。

## フォルダ構成

```
<ピクチャ>/CheckListMaker/                  ← 親フォルダは従来と同じ
  <リスト名>_<yyyymmdd>_<hhmmss>/           ← 録画1回 = 1セッションフォルダ
    session.json                            ← セッション全体のメタデータ
    001.png                                 ← 撮影画像（従来同様マーカー焼き込み済み）
    001.json                                ← 1クリック分のサイドカー
    002.png
    002.json ...
```

- セッションフォルダは**録画開始（ガジェットの「録画開始」ボタン＝ `rec:begin`）時**に
  作成する。同名フォルダが既にある場合は `_2`, `_3`… を付けて衝突を回避する。
- 連番は**セッション内で 1 から**。3桁ゼロ埋め（`001`〜。1000枚以上は自然に4桁）。
  フォルダがセッションごとに分かれるため、旧方式の「同日ファイルの最大番号+1」走査
  （`nextSequence`）は不要になり削除する。
- **1枚も撮らずに停止した場合、空のセッションフォルダは削除する**（ゴミを残さない）。
- 録画停止時に開くフォルダは**そのセッションのフォルダ**（0枚時は従来どおり親フォルダ）。
  ガジェットのプレビュークリック（`rec:openDir`）も録画中はセッションフォルダを開く。
  **2026-07-13 更新（2-R4）**: 停止時の自動オープンは取り込みウィザードの表示に置き換え、
  フォルダはウィザード内の導線から開く。→ `docs/spec-2-R4-import-wizard.md`
- 旧方式のフラットなファイル（`<リスト名>_<日付>_<連番>.png`）はそのまま残る。
  マイグレーションはしない（撮影済み画像は完成した成果物であり、形式変換の利益がない）。

## session.json（セッション単位のメタデータ）

```json
{
  "version": 1,
  "type": "checklistmaker-recording",
  "name": "サーバー点検",
  "startedAt": "2026-07-13T05:12:34.567Z",
  "endedAt": "2026-07-13T05:15:02.001Z",
  "shots": 12
}
```

- 録画開始時に `endedAt: null` で書き、撮影のたびに `shots` を更新、停止時に
  `endedAt` を確定する。**`endedAt` が null のまま残っていれば異常終了の痕跡**になる
  （R4 の取り込みウィザードが「未完了セッション」を検出できる）。
- R4 はフォルダ名を解析せず `session.json` からリスト名・日時を読む。

## NNN.json（1クリック分のサイドカー）

```json
{
  "version": 1,
  "seq": 3,
  "image": "003.png",
  "time": "2026-07-13T05:13:10.123Z",
  "elapsedMs": 35556,
  "click": { "button": "left", "clicks": 1, "x": 2418, "y": 507 },
  "imagePoint": { "x": 498, "y": 507 },
  "display": {
    "id": 2779098405,
    "boundsDip": { "x": 1920, "y": 0, "width": 1920, "height": 1080 },
    "scaleFactor": 1
  },
  "marker": { "drawn": true, "x": 498, "y": 507, "radius": 20, "lineWidth": 4, "color": "#ef4444" },
  "capture": { "source": "precapture" },
  "uia": {
    "resolved": false,
    "name": null,
    "controlType": null,
    "rect": null,
    "windowTitle": null,
    "appName": null
  }
}
```

| フィールド | 内容 |
| --- | --- |
| `time` / `elapsedMs` | 保存時刻（ISO 8601 UTC）と録画開始からの経過ms |
| `click` | `button`: `"left"`/`"right"`。`clicks`: uiohook の連続クリック数（mouseup 時点の値）。`x`/`y`: グローバル物理px座標 |
| `imagePoint` | **撮影画像内**のクリック位置（物理px。画像の左上原点）。R3 の切り出し中心はこれを使う |
| `display` | Electron の Display 情報。`boundsDip` は DIP 座標（uiohook の物理pxとは `scaleFactor` 倍違う点に注意） |
| `marker` | 赤丸を焼き込んだか。焼き込み時は画像内座標・実効半径（`scaleFactor` 乗算後）・線幅・色。R3 が「旧式マーカーが写っている領域」を知るために使う |
| `capture.source` | `"precapture"`（事前キャプチャ採用）か `"ondemand"`（押下時撮影） |
| `uia` | **R2 で埋める枠**。R1 時点では `resolved: false`・全項目 null の雛形のみ |

> **2026-07-13 更新**: 2-R2 でスキーマ **v2** となり、`uia` に実データが入り
> トップレベルに生成文 `text` が加わった。→ `docs/spec-2-R2-uia-steptext.md`

### 既知の制限（R2b で扱う）

- **ダブルクリック**: 撮影デバウンス（連打を1枚に集約）により、2回目のクリックは
  保存されない。サイドカーの `clicks` は「保存を確定した mouseup 時点」の値のため、
  ダブルクリックでも通常 `1` が記録される。連続クリックの認識と「ダブルクリック」文の
  生成は **R2b の必須要件**として、操作認識を撮影と独立に記録する方式で対応する
  （2-R0 で `clicks` フィールドの有効性は実証済み）。
- ドラッグは従来どおり記録しない（R2b のオプション）。

## エラー方針

- **PNG の書き込み失敗は従来どおり**（保存チェーンの catch → ガジェットへ警告）。
- **サイドカー・session.json の書き込み失敗は撮影を止めない**（console.error のみ）。
  画像が主成果物であり、メタデータ欠落は取り込み時にフォールバックできるため。
- セッションフォルダの作成に失敗したら、録画は継続するが各撮影が保存失敗として
  ガジェットに警告される（従来の保存先フォールバック経路と同じ挙動）。

## 実装配置

- **`session.js`（新規・メインプロセス）**: セッションフォルダの作成・連番採番・
  サイドカー/session.json の書き出し・終了処理。録画エンジン v2 のファイル分割
  （ロードマップ前提2）の最初のモジュール。
- **`main.js`**: `persistShot()` が `session.js` へ書き出しを委譲。`nextSequence` /
  `dateStamp` / `escapeRegExp`（旧フラット採番）は削除。mousedown/mouseup から
  `button` / `clicks` / 撮影ソースをサイドカー用に引き回す。
- **`test/session.test.js`（新規）**: フォルダ作成・衝突回避・連番・サイドカー内容・
  空セッション削除・終了処理の単体テスト（node:test。Electron 不要）。
