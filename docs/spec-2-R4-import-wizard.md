# 2-R4 取り込みウィザード — 確定仕様

ロードマップ **2-R4** の実装前に確定した仕様。録画セッション（2-R1〜R3 の素材:
全景・拡大・生成文・メタデータ）をアプリ内で**並べ替え・削除・統合・文言調整**して、
チェックリストの新セクションとして取り込む。**R1〜R4 完了＝中間ゴール
（SaaS対抗ライン）の成立点**（前提8）。

## ユーザー確定事項（2026-07-13）

1. **録画停止時はエクスプローラーではなく取り込みウィザードを開く。**
   停止するとアプリ本体が前面に戻り、そのセッションのウィザードが自動で開く。
   フォルダはウィザード内の「フォルダを開く」ボタンから開ける（自動では開かない）。
2. **過去のセッションも一覧から選んで取り込める。** リスト編集画面に
   「録画を取り込む」導線を追加し、ピクチャ配下のセッション一覧（新しい順・
   未完了／取り込み済みバッジ付き）から選んで同じウィザードを開く。
   異常終了したセッション（`endedAt: null`）も取り込める（リカバリー導線）。
3. **各ステップに添付する画像は「どちらか適切な方」を自動選択する。**
   - 要素矩形が採用できたステップ（`zoom.source === "element"`）→ **拡大**
     （枠付きで箇所が読める）
   - それ以外（クリック中心フォールバック・拡大なし・旧形式 v1/v2）→ **全景**
     （フォールバック文「図の位置をクリック」は赤丸の写った全景とセットで読む文言のため）
   - ウィザードでステップごとに **拡大／全景／両方** に切り替えられる。

## 全体フロー

```
録画停止 ──→ rec:done イベント ──→ ウィザード表示（自動）
編集画面「録画を取り込む」──→ セッション一覧 ──→ ウィザード表示（手動）

ウィザード: ステップ一覧（サムネイル＋文）を編集
  並べ替え(↑↓) / 削除(✕) / 上と統合 / 画像切替(拡大/全景/両方) /
  文の個別編集 / 一括置換（検索→置換） / セクション名 / 取り込み先リスト選択
    ↓ 取り込む
対象リストの末尾に新セクション（既定名「録画 yyyy/MM/dd HH:mm」）として追加
  ・1ステップ = 1項目（text = 生成文または編集後の文）
  ・画像は既存の取り込みパイプライン（compressImage → 1600px JPEG）で添付
  ・store.commit 1回 = Undo 1回で取り消せる
    ↓
session.json に importedAt を記録（取り込み済みバッジの根拠）
```

- 0枚のセッションは停止時にフォルダごと削除される（2-R1）ため、
  ウィザードは開かずトーストで知らせるだけ。
- ブラウザ単体モード（`recorderAPI` なし）では導線を出さない（録画ボタンと同じ扱い）。

## session.json の拡張（バージョン 2）

```json
{
  "version": 2,
  "type": "checklistmaker-recording",
  "name": "サーバー点検",
  "startedAt": "...", "endedAt": "...", "shots": 12,
  "importedAt": "2026-07-13T06:00:00.000Z"
}
```

- `importedAt`: 取り込み完了時刻（未取り込みは null）。同じセッションを再度
  取り込むことは禁止しない（バッジで注意を促すだけ。二重取り込みは Undo で戻せる）。
- v1（importedAt なし）は「未取り込み」として扱う。読み取りは後方互換。

## メインプロセス API（session.js に追加・main.js が IPC で公開）

| 関数 / IPC | 内容 |
| --- | --- |
| `listSessions(parentDir)` / `rec:sessions` | 直下のセッションフォルダ（session.json を持つもの）を新しい順に列挙。`{ dir, name, startedAt, endedAt, shots, importedAt }` の配列 |
| `readSession(dir)` / `rec:session` | セッション1件の読み込み。`NNN.png` をスキャンし、サイドカーを併読して `{ info, steps: [{ seq, image, zoomImage, text, uia, click, time }] }` を返す。サイドカー欠落・壊れは text 等 null のステップとして返す（画像が主成果物） |
| `markImported(dir)` / `rec:markImported` | session.json に importedAt を記録（read-modify-write。他フィールドは保持） |
| — / `rec:image` | 画像1枚を dataURL で返す（サムネイル表示・添付用） |
| — / `rec:openDir`（拡張） | 引数 dir 指定時はそのセッションフォルダを開く |

- **パス検証**: IPC が受け取る `dir` は「ピクチャ配下の `CheckListMaker` の直下」に
  解決されること、画像ファイル名は `NNN.png` / `NNNz.png` 形式のみ許可
  （レンダラー侵害時に任意ファイルを読ませないため）。

## ウィザード UI（index.html・buildModal ベース）

- **ステップ行**: サムネイル（選択中の画像・遅延読み込み・320px に縮小してキャッシュ）
  ＋ 連番 ＋ 文（textarea・個別編集可）＋ 画像切替（拡大/全景/両方。拡大が無い
  ステップは拡大を選べない）＋ ↑ / ↓ / 上と統合 / ✕（削除）。
- **統合**: 対象ステップを直前のステップへ結合する。文は「、」で連結
  （体言止めの連続として読める）。画像はそれぞれの選択を保持したまま両方添付される。
- **一括置換**: 検索語 → 置換語を全ステップの文へ適用（単純文字列置換）。
  「文言の一括調整」のロードマップ要件に対する最小実装。自然文化・丁寧語化は
  4-R7（ローカルLLM）の領域であり本工程では扱わない。
- **フッター**: 取り込み先リストの選択（既定 = 録画開始時に開いていたリスト。
  無ければ現在開いているリスト）＋ セクション名（既定「録画 yyyy/MM/dd HH:mm」・
  編集可）＋「取り込む」。
- 取り込み中は進捗をボタンに表示（画像の読み込み・圧縮があるため）。

## 画像の添付

- 添付時は既存の `compressImage`（長辺 1600px・JPEG 品質 0.85）を通す。
  既存の取り込み画像と同じ扱いになり、1-1 の参照化・3-B の原寸保存方針にもそのまま乗る。
- 「両方」の並びは **全景 → 拡大**（場所 → 詳細の読み順）。
- 元 PNG はセッションフォルダに残る（消さない）。取り込みは何度でもやり直せる。

## レンダラー純関数（テスト対象・`window.__test__` に公開）

| 関数 | 内容 |
| --- | --- |
| `wizardDefaultChoice(step)` | 既定画像の自動選択（element→'zoom'、それ以外→'full'） |
| `wizardStepsFrom(sessionData)` | readSession の結果 → ウィザードのステップモデル |
| `wizardMergeUp(steps, index)` | index を直前へ統合（文の連結・shots の結合） |
| `wizardBulkReplace(steps, find, replace)` | 全ステップの文へ一括置換 |
| `buildImportSection(title, steps, imagesByStep)` | ステップモデル → セクション（items＋images） |

## エラー方針

- セッション一覧・読み込みの失敗はトーストで通知し、ウィザードは開かない。
- 画像1枚の読み込み・圧縮失敗はそのステップの画像だけ諦めて続行
  （文は取り込む。console.error 記録）。
- importedAt の書き込み失敗は取り込み結果に影響させない（console.error のみ）。

## スコープ境界（この工程でやらないこと）

- 文章の自然文化・丁寧語化・要約（→ 4-R7 ローカルLLM）。
- ダブルクリック等の操作認識と文への反映（→ 2-R2b）。
- ステップのドラッグ&ドロップ並べ替え（↑↓ボタンで代替。要望があれば後続で）。
- 取り込み後のセッションフォルダ自動削除（素材は残す。掃除はユーザーの領分）。
