# 仕様: 手順本文への表挿入（リッチ本文）機能

テンプレート型の手順欄に、文章と表を任意の位置に配置できる「リッチ本文」を追加する。
本ドキュメントはユーザーとの合意事項を確定仕様として記録する（運用ルール 領域1）。

## 1. 対象・データモデル

- **対象**: テンプレート型の手順のみ。ToDo 型は従来通り（変更しない）。
- 手順は次の3要素構成になる:
  1. **見出し行** … 従来の1行テキスト（`item.text`）。そのまま維持。
  2. **リッチ本文** … 新設。文章＋表を任意位置に混在配置できる。
  3. **メモ（任意）** … 従来の `item.note`。残す（本文＝説明・表、メモ＝補足、と役割分担）。
- リッチ本文は item に新フィールド `item.body` として **サニタイズ済み HTML 文字列** で保持。
  未設定（空）でも動作する＝既存データと後方互換。

## 2. リッチ本文でできること

- **書式付きテキスト**（本文・表セルの両方で共通）:
  - 太字 / 斜体 / 下線 / 取消線
  - 文字色
  - ハイライト（背景色）
  - 箇条書き / 番号付きリスト
- **表の挿入**: 本文中の任意位置に埋め込み。
  - **先頭行を見出し**（太字＋地色で強調）。
  - **行・列はボタンで自由に追加/削除**。
  - **セルの中身も書式付きテキスト**（同じ装飾が使える）。
  - **セル塗りつぶし**（セルの背景色）。文字のハイライトとは別に、セル単位で地色を付ける。
  - **列幅は原則自動調整**（内容に応じて自動）。
  - **セル内改行は Alt+Enter**（通常の Enter はセル/段落の確定・移動に使う）。
- **スプレッドシート貼り付け取り込み**: Excel / Google スプレッドシートからコピーした表を
  貼り付けるとセルへ一括取り込み（手動作成と併用）。

## 3. 編集 UI

- **常設ツールバー**: 本文編集中（フォーカス時）に上部へ表示。装飾ボタン＋「表を挿入」ボタン。
- **インライン編集**: 本文・セルを画面上で直接クリック編集（contentEditable）。
- **行/列操作**: 表にカーソルを近づけたとき**だけ**縁にコントロールを表示する（常時は非表示）。
  - **＋（追加）は端（角）に配置**: 列追加＝右上の角、行追加＝左下の角。
  - **−（削除）は対象の列/行の中央**: 列削除＝各列の上（列幅の中央）、
    行削除＝各行の左（行高の中央）。列・行で配置を対称にする。

## 4. 出力・その他

- **印刷 / PDF**: リッチ本文と表（罫線・見出し行）をそのまま出力。
- **共有リンク**: 表・装飾はテキストなので同梱可（リンクはやや長くなる）。
  画像は従来通り共有リンクには非同梱。

## 5. 実装上の留意点（決定事項ではなく注意）

- **HTML サニタイズ必須**: contentEditable とスプレッドシート貼り付けは余計なタグ/スタイルが
  混入するため、許可タグ・スタイルのホワイトリストで保存時・描画時に浄化する（XSS 対策）。
  - 許可タグ例: `p, br, strong/b, em/i, u, s, span, ul, ol, li, table, thead, tbody, tr, th, td`
  - 許可スタイル例: `span` の `color` / `background-color`、`td/th` の `text-align` と
    `background-color`（セル塗りつぶし）程度（列幅は自動のため width は基本保持しない）
- 装飾は本体既存の系統（画像エディタでも使う execCommand 系）で実装可能。Electron/Chromium 動作。
- Alt+Enter は keydown で捕捉し、セル/段落内に `<br>` を挿入する独自ハンドラで実装。

## 6. 進め方

1. 本仕様ドキュメントを作業ブランチ `claude/step-rich-body-table` に記録（本コミット）。
2. 編集画面・本文/表の見た目の **モックイメージ** を出力してデザイン確認。
3. 承認後に実装 → 検証 → push。PR 作成・マージはユーザー承認を待つ（領域3）。
