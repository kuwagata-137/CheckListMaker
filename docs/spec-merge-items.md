# 仕様: 手順/項目の「上と統合」（通常編集）

取り込みウィザードにあった「統合」を、**通常の編集画面でも**使えるようにする。
本ドキュメントはユーザーとの合意事項を確定仕様として記録する（運用ルール 領域1）。

- 決定日: 2026-07-16（ユーザー確定「これでOK」）
- 実装: `index.html`（model `mergeItemUp`/`mergeItemInto`/`mergeTimeValue`、UI ボタン、
  pointerdown ハンドラ）。テストは `test/merge.test.js`。

## 導線

- 各手順/項目の操作アイコン列（↑ ↓ の隣、✕ の前）に「**⤒ 上の手順と統合**」を追加。
- 押すと、その手順を**1つ上の手順に統合**（取り込みウィザードの「上と統合」と同じ方向）。
- **セクションの先頭手順は無効**（`disabled`＝グレーアウト。統合先が無いため）。
- 1操作＝1コミットで、**Ctrl+Z（Undo）1回で元に戻せる**。
- 押下前に編集中の入力欄を blur して確定させる（pointerdown 経路を流用）。

## フィールドの統合ルール（確定）

| フィールド | 統合のしかた |
| --- | --- |
| 手順テキスト `text` | 「、」で連結。片方が空ならもう片方をそのまま（区切りを入れない） |
| 本文（リッチ）`body` | 上の本文の後ろに下の本文を連結（段落 HTML として結合。片方空はもう片方） |
| メモ `note` | 改行 `\n` で連結（片方空はもう片方） |
| 所要時間 `time` | 合算（両方数値のときだけ足す。片方だけ数値ならその値。どちらも非数値なら空） |
| 画像 `images` / `imagesFull` / `imageEdits` | 並行3配列を **images 長に正規化してから連結**し、index 対応（サムネ⇔原寸⇔編集ソース）を保つ |
| 完了 `done` | **両方チェック済みのときだけ**チェック済み（どちらか未完なら未完＝安全側） |

## 適用範囲（確定）

- **対象**: テンプレート型の手順・ToDo 型の項目の**両方**（ToDo は body/note/time が
  空のままなので実質テキストと画像だけ統合される）。
- **セクション境界**: **同一セクション内のみ**。別セクションの先頭手順を上のセクション
  末尾へは統合しない。
- **方向**: 「上と統合」のみ（「下と統合」は設けない）。

## 実装メモ

- `mergeItemInto(prev, cur)`（純関数）が cur を prev に吸収させる。並行配列は
  `itemImages`/`itemImageEdits`/`itemImagesFull` で正規化してから `concat` する。
- `mergeItemUp(checklist, sectionId, itemId)` は同一セクション内で対象の直前へ統合し、
  対象（下側）を取り除く。先頭・不明 id は no-op。
- 取り込みウィザードの `wizardMergeUp`（録画ステップ用・text と shots のみ）は別物として
  そのまま残す（統合対象のデータ構造が異なるため）。
